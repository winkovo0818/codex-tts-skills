# Douyin Novel TTS QA Checklist

Use this checklist before delivering a short-video adaptation package.

## Story

- The first 3 seconds contain a specific conflict, secret, death, betrayal, humiliation, deadline, or reversal.
- The protagonist's identity and desire are understandable without reading prior chapters.
- Each episode has one main conflict beat.
- The cliffhanger is concrete, not just "what happens next?"
- No invented fact breaks the chapter's continuity.

## Oral Script

- Sentences sound natural when read aloud.
- Long exposition is compressed into action, dialogue, or consequence.
- Names and relationships are not confusing.
- TTS pause/emotion notes are sparse and useful.
- The script can stand alone as audio.

## TTS Package

- Title and cover text sell the same core conflict.
- The spoken text can be copied directly into a TTS tool.
- No visual directions are included unless explicitly requested.
- The next TTS production skill can parse each episode's口播稿 cleanly.
- AI/TTS usage is disclosed or label-ready where platform rules require it.

## Testing

- At least 3 alternate hooks are provided for episode 1.
- One variable is recommended for A/B testing first.
- Metrics include 3-second hold, completion, comments asking for updates, saves, and follows.
